# Расчёт КПД парового котла — обратный баланс

Независимая реализация нормативной методики теплового расчёта котельных агрегатов
(Кузнецов Н.В. и др., ВТИ/ЦКТИ) с графическим интерфейсом на Python/tkinter.

Восстановлена и валидирована при реверс-инжиниринге старой DOS-программы расчёта
КПД котла (Turbo Basic, 1987) — формулы q2–q6, Qp, B сверены с опубликованным
источником и проверены на реальных исторических данных котельной.

Инструкция по заполнению полей для каждого вида топлива — см. **[USAGE.md](USAGE.md)**.

## Запуск из исходников

Нужен только Python 3.10+ (стандартная библиотека, tkinter):

```
python3 kpd_gui.py
```

## Сборка отдельного .exe для Windows

Через GitHub Actions (вкладка **Actions** → **Build Windows exe** → **Run workflow**),
либо локально на Windows:

```
pip install pyinstaller
pyinstaller --onefile --windowed --name KPD_Kotla kpd_gui.py
```

Готовый файл: `dist/KPD_Kotla.exe` — не требует установки Python на целевой машине.

## Файлы

- `kpd_gui.py` — графический интерфейс (точка входа)
- `kpd_core.py` — расчётное ядро (формулы теплового баланса)
- `fuel_data.py` — база топлив (20 сортов угля, 3 мазута, газовые топлива)
- `batch_validate.py` — сверка расчёта с историческими данными (.ods-протоколы 2015-2020)
- `precise_validate.py` — сверка с точным (3 знака) выводом оригинальной DOS-программы

## Точность и нормативы допустимой погрешности

### Нормативная допустимая погрешность (тепловые испытания котлов)

| Категория испытаний | Допустимая погрешность КПД |
|---|---|
| I/II категория (определительные испытания) | ±1% (абс.) |
| II категория сложности | 1.5–2% |
| III категория / эксплуатационные испытания | 2–3.5% |
| ГОСТ Р 54442-2011 (суммарная погрешность измерений) | ±2% |

### Погрешность этой программы относительно оригинальной DOS-версии (Kpd3t2.exe)

**Точечная проверка** (все входные данные контролируются вручную, живой прогон
оригинальной программы в DOSBox с известными точными входами):

> КПД(наша программа) = 83.196% против КПД(DOSBox) = 83.104% → **расхождение 0.11%**

**Сверка по 7 парам** (вход из реального `.ods`-протокола → наш расчёт, сравнение
с точным выводом DOSBox-версии, 3 знака после запятой, `precise_validate.py`):

> медиана |Δ| = 2.41 п.п., среднее |Δ| = 2.56 п.п., диапазон от −3.21 до +3.78 п.п.

**Массовая сверка по 36 файлам** архива 2015-2020 (`batch_validate.py`):

> медиана |Δ| = 2.32 п.п., среднее Δ = +2.11 п.п., 36% результатов — в пределах ±2 п.п.

**Вывод:** при полностью известных входных данных программа воспроизводит оригинал
с точностью 0.11% — заметно лучше норматива I категории (±1%). Разброс 2-3 п.п. на
историческом архиве укладывается в норматив III категории/эксплуатационных испытаний
(2-3.5%), но не в строгую I/II категорию — и объясняется не ошибкой формул, а тем, что
в сохранившихся протоколах нет полного элементного состава угля на каждую пробу (только
влажность и зольность) и есть неоднозначность, по какой из трёх точек замера O2 в тракте
котла (перед скруббером / перед дымососом / за пароперегревателем) в своё время считала
оригинальная программа.

## Источники и авторство

Код написан с помощью [Claude Code](https://claude.com/claude-code) (Anthropic).

Формулы теплового баланса и табличные данные топлив (состав рабочей массы, низшая
теплота сгорания, объёмы продуктов сгорания, энтальпии) взяты из учебной и нормативной
литературы:

- **Кузнецов Н.В. и др.** «Тепловой расчёт котельных агрегатов. Нормативный метод» —
  ВТИ/ЦКТИ, Энергия, 1973.
- **Смородин С.Н. и др.** «Тепловой и аэродинамический расчёты котельных агрегатов» —
  учебное пособие, СПбГТУРП.

Программа не является официальной публикацией указанных источников и не заменяет их —
это независимая инженерная реализация описанной в них методики.
